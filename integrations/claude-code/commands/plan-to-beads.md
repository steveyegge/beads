# Convert Plan to Beads Tasks

## description:
Convert a Claude Code plan file into beads epic + tasks for cross-session tracking.

## Arguments
$ARGUMENTS (optional - path to plan file, defaults to most recent in ~/.claude/plans/)

---

Use the **Task tool** with `subagent_type='general-purpose'` to convert the plan.

## Agent Instructions

The agent MUST operate with defensive execution. Do not assume success.

1. **Find and Validate the plan file**
   - If argument provided, use that path.
   - Otherwise: `ls -t ~/.claude/plans/*.md | head -1`.
   - **Verification:** If the file is empty or missing, STOP and report the error.

2. **Parse the plan structure**
   - Title: First `# Plan:` or `#` heading.
   - Description: Content under `## Summary`.
   - Tasks: Each `### Phase N:` or `### N.` section.

3. **Create the epic with Result Verification**
   - Execute: `bd create "[Plan Title]" -t epic -p 1 -d "[summary]" --json`
   - **CRITICAL:** Capture the JSON output. If the command fails or times out (30s), DO NOT proceed. Parse the `id` from the JSON. If no `id` is present, report the `stderr` to the user.

4. **Create tasks from phases (Atomic Operations)**
   - For each phase, run the `bd create` command with `--json`.
   - **Verification:** You MUST extract the real UUID/ID from the command output before attempting to create dependencies. Never hallucinate a task ID.

5. **Sequential Dependencies & Linking**
   - Only execute `bd dep add` once you have verified the IDs of both the parent and child tasks.
   - If a dependency link fails, retry once. If it fails again, note the break in the final summary.

6. **Return a Verifiable Summary** (not raw output):
   - Include the real IDs generated by the system.
   - If any step failed or timed out, explicitly list it as "FAILED" with the reason (e.g., "Subprocess Timeout" or "API 5xx").

7. **Return a concise summary** (not raw output):
   ```
   Created from: [filename]

   Epic: [title] ([epic-id])
     ├── [Phase 1] ([id]) - ready
     ├── [Phase 2] ([id]) - blocked by [prev]
     └── [Phase 3] ([id]) - blocked by [prev]

   Total: [N] tasks
   Run `bd ready` to start.
   ```

## Notes

- Original plan file is preserved for reference
- Task descriptions use first paragraph only (keeps them scannable)
- Sequential phases get automatic dependencies
